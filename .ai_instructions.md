# Instruções de Contexto - LexiCard PWA (Learning App)

Este documento serve como a única fonte de verdade para as regras de negócio, padrões de código e design do projeto. A IA deve consultar estas regras antes de cada sugestão ou geração de código.

## 1. Stack Tecnológica
- **Framework:** Expo (React Native for Web) para suporte a PWA, Android e iOS.
- **Linguagem:** TypeScript (Tipagem estrita obrigatória).
- **Estilização:** NativeWind (Tailwind CSS para React Native).
- **Banco de Dados:** Supabase (PostgreSQL + Auth + SDK).
- **Gerenciamento de Estado:** React Context API ou Zustand.
- **Persistência Local:** AsyncStorage (para suporte offline).
- **API Externa:** dictionaryapi.dev (Fonte primária de dados).

## 2. Regras de Negócio (Flashcards)
- **Fluxo de Dados:** 1. Verificar cache local (AsyncStorage).
    2. Se não existir, verificar Supabase.
    3. Se não existir, consultar `dictionaryapi.dev`, exibir e salvar no Supabase/Local.
- **Interface do Card:**
    - **Frente:** Exibe a palavra principal em Inglês.
    - **Interações na Frente:**
        - **Ícone de Áudio:** Reproduz a pronúncia (MP3) via URL da API. *Nota: Funciona apenas online.*
        - **Ícone de Exemplo:** Ao clicar, exibe a definição e o exemplo da frase.
        - **Ícone de Tradução:** Gatilho visual para a tradução.
    - **Clique Central:** Vira o card (flip) para revelar a tradução em Português.
    - **Ações de Feedback:** Após o flip, exibir botões de "Acerto" (Check) e "Erro" (X).
- **Gestão de Mídia:** Armazenar apenas a URL do áudio no Supabase e no AsyncStorage. Não realizar download do ficheiro binário.

## 3. Padrões de Código e Arquitetura
- **Componentes:** Funcionais com Hooks. PascalCase para ficheiros.
- **Tipagem:** Interfaces TypeScript para todas as respostas da API e estruturas de dados.
- **Clean Code:** - Nomes de variáveis em camelCase e descritivos.
    - Funções pequenas e com responsabilidade única.
    - Proibido o uso de `any`.
- **Organização de Pastas:**
    - `/src/components`: Componentes reutilizáveis.
    - `/src/services`: Chamadas de API e Supabase.
    - `/src/hooks`: Lógica de negócio extraída em hooks customizados.
    - `/src/store`: Configurações de estado e persistência.

## 4. Guia de Estilo (UI/UX)
- **Paleta de Cores:**
    - Primária: `#4F46E5` (Indigo)
    - Fundo: `#F8FAFC` (Cinza claro)
    - Sucesso: `#10B981` (Esmeralda)
    - Erro: `#EF4444` (Vermelho suave)
    - Texto: `#1E293B` (Slate escuro)
- **Tipografia:**
    - Títulos: Inter_700Bold
    - Corpo: Inter_400Regular
- **Design:** Bordas arredondadas (rounded-xl), sombras suaves (shadow-sm) e aspecto "clean" de aplicação de educação moderna.

## 5. Comportamento da IA
- **Zero Delírios:** Se a biblioteca sugerida não for compatível com Expo ou PWA, não a utilize.
- **Segurança:** Nunca exponha chaves de API diretamente no código; use variáveis de ambiente (.env).
- **Profissionalismo:** O código será usado em portfólio no LinkedIn. Deve seguir as melhores práticas de produção.
